Flight Deck Technology Stack

Reference documentation for the core platform products powering the Fiserv Internal Developer Platform

💻
Cloud Dev Environment

Coder

Self-hosted cloud development environments that enable consistent, secure, and instant developer workspaces across any infrastructure.

  • coderd — Central API layer managing dashboard, HTTP API, dev URLs, and agent registration with PostgreSQL state storage
  • provisionerd — Infrastructure provisioning via Terraform with support for multiple daemon instances
  • Workspaces — Collections of cloud resources (VMs, Kubernetes, storage) with persistent and ephemeral resource types
  • Agents — Cross-platform agents providing SSH access, port forwarding, liveness monitoring, and startup automation
  • Flexible Deployment — Bundled or decomposed architecture for latency reduction and horizontal scaling
  • Enterprise Integrations — GitHub Enterprise, GitLab, Bitbucket, JFrog Artifactory, and private registries

Internal Developer Portal

Cortex.io

AI-powered Internal Developer Portal that helps engineering leaders build organizations that ship reliable, secure, and efficient software, faster.

  • Service Catalog — Centralized metadata for services, APIs, pipelines, and resources with 60+ out-of-box integrations
  • Scorecards — Custom scorecards via CQL to gauge operational readiness, DORA metrics, security compliance, and development maturity
  • Developer Self-Service — Pre-approved templates and composable workflows for autonomous service provisioning
  • Engineering Intelligence — DORA metrics, scorecard trends, and portfolio-level analytics dashboards
  • Leadership Dashboards — Executive reports with initiative tracking and compliance monitoring
  • AI Integration — Compatible with Claude Code, Cursor, VSCode, and AI-powered service ownership prediction
🌐 Product Overview 📖 Documentation 💻 Demo Portal
⚙
Platform Orchestrator

Kratix

Open-source platform orchestrator that enables platform teams to deliver self-service, secure, and scalable platform capabilities as governed APIs.

  • Promises — Declarative contracts defining platform capabilities as self-service APIs with embedded workflows
  • Self-Service APIs — Immediate and autonomous resource provisioning through Promise-defined interfaces
  • Embedded Workflows — Consistent compliance, security, and configuration enforcement within each Promise
  • Fleet Management — Built-in reconciliation keeping resources up-to-date across multi-cluster deployments
  • Promise SDKs — Python and Golang SDKs for rapid Promise development with testing frameworks
  • Terraform Integration — Convert existing Terraform modules to managed services via "kratix init" CLI

CD Practices & Standards

Minimum CD

Community-driven definition of the minimum practices required for Continuous Delivery, with an interactive tool to assess and track adoption of CD capabilities.

  • Core CD Practices — Trunk-based development, continuous integration, automated testing, and build-on-commit fundamentals
  • Deployment & Release — Deployment automation, feature flags, on-demand rollback, and immutable artifacts
  • Testing Strategy — Contract testing, integration testing, security testing, performance testing, and deterministic tests
  • Infrastructure & Config — Infrastructure automation, configuration management, secret management, and ephemeral environments
  • Observability — Telemetry, monitoring & alerting, logging infrastructure, and pipeline visibility
  • Team Practices — Cross-functional product teams, component ownership, stop-the-line culture, and unified backlogs

⚡ Flight Deck Platform Architecture

The Flight Deck platform integrates three core products to deliver a unified developer experience. Cortex serves as the single pane of glass providing visibility, scorecards, and engineering intelligence. Coder enables instant, consistent cloud development environments integrated with the service catalog. Kratix orchestrates multi-cluster deployments through declarative Promises, delivering golden path infrastructure with Harness CD integration.
